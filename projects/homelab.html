Overview

What I Built

A multi-VM environment running on VMware Workstation that mirrors a small enterprise network. The goal was hands-on repetition with the systems I'd encounter in a real IT or security role — in a place where breaking things is expected.

  • Deployed and configured Windows Server 2019 as a domain controller
  • Set up Active Directory: users, groups, OUs, and group policy
  • Configured DNS, DHCP, and internal name resolution
  • Joined Windows client machines to the domain
  • Introduced controlled misconfigurations and practiced remediation
Lab Stack

Environment

  • VMware Workstation — virtual network host
  • Windows Server 2019 — AD DS, DNS, DHCP
  • Windows 10/11 — domain-joined client
  • Kali Linux — offensive security tooling
  • Internal virtual network with isolated segments
Active Directory DNS Group Policy Firewalls
Security Work

What I Practiced

  • Firewall rules and baseline hardening
  • User privilege escalation and access control testing
  • Log review and basic incident analysis
  • Deploying and using Kali security tools in the lab
Takeaways

What I Learned

  • How Active Directory actually structures an enterprise environment
  • Why network segmentation matters for security
  • How to diagnose and fix real infrastructure problems
  • The value of breaking things in a safe place to understand how they work

Why This Project Matters

Most IT and security skills are best learned by doing, not reading. This lab is how I build real, hands-on experience with the infrastructure and security concepts that matter most — before stepping into a professional environment.
